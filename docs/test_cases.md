# Siren テストケース（大枠）

Siren プラグインの品質を確保するための、ハイレベルなテストケース構成案です。

## 1. 言語サポート (Language Support)
エディタ上での入力補助機能が正しく動作することを確認します。

- **構文ハイライト**:
    - 各種ダイアグラム（flowchart, sequenceDiagram等）のキーワードが正しく色分けされること。
    - コメント（`%%`）、矢印（`-->`）、括弧（`[]`, `{}`, `()`）がハイライトされること。
- **コード補完**:
    - ファイルの先頭でダイアグラムタイプの補完候補が表示されること。
    - 各ダイアグラム固有のキーワード（`subgraph`, `participant`等）が適切な文脈で補完されること。
- **ライブテンプレート**:
    - 定義された略称（`flowchart`等）を入力して展開した際、正しい雛形が挿入されること。
- **コメント機能**:
    - 行コメントの切り替え（Ctrl+/ / Cmd+/）が正しく動作し、`%%` が付与/除去されること。
- **括弧のペアリング**:
    - 括弧入力時に閉じ括弧が自動挿入されること。
    - 対応する括弧が強調表示されること。

## 2. プレビューとレンダリング (Preview & Rendering)
ダイアグラムが正しく描画され、操作できることを確認します。

- **リアルタイム更新**:
    - エディタでの変更が、一定の遅延または即座にプレビュー画面に反映されること。
- **レンダリングの正確性**:
    - サポート対象の全24種類のダイアグラムが、エラーなく表示されること。
    - 構文エラーがある場合に、プレビュー側で適切にエラー表示（または以前の表示を維持）されること。
- **ズーム機能**:
    - ツールバーのズームイン/ズームアウトが動作し、表示サイズが変わること。
    - ズーム後も描画が鮮明であること。
- **テーマ対応**:
    - IDEのテーマ（Dark/Light）に合わせて、プレビューの背景やダイアグラムの色味が適切に調整されること。

## 3. Markdown 連携 (Markdown Integration)
標準の Markdown プラグイン内での動作を確認します。

- **コードブロックのレンダリング**:
    - Markdown 内の ` ```mermaid ` ブロックが、プレビュー上でダイアグラムとして描画されること。
- **拡張機能の有効化**:
    - Markdown プレビュー拡張として Siren が正しく認識・ロードされていること。
- **ダークモード検知**:
    - Markdown プレビュー内の Mermaid ダイアグラムが、プレビュー全体のテーマ設定に従うこと。
- **複数の Mermaid ブロック**:
    - 1 つの Markdown ファイル内に複数の ` ```mermaid ` ブロックがある場合、それらすべてが正しくレンダリングされること。
- **動的更新**:
    - Markdown エディタでの変更が、Markdown プレビュー上の Mermaid ダイアグラムに即座に（または適切なデバウンス後に）反映されること。

## 4. システムと統合 (System & Integration)
プラグインの基盤部分が安定して動作することを確認します。

- **ファイル認識**:
    - `.mermaid` および `.mmd` 拡張子のファイルが Mermaid 言語として認識されること。
- **サーバーライフサイクル**:
    - エディタが開かれた際に内部 HTTP サーバーが起動し、閉じられた際に停止すること。
    - ポート競合時に、自動的に空きポートを探して起動できること。
- **JCEF ブラウザ**:
    - プレビューウィンドウ（JCEF）が正しく初期化され、リソース（mermaid.min.js等）をロードできること。

## 5. 境界条件・異常系 (Boundary & Edge Cases)
極端な条件下での挙動を確認します。

- **巨大なダイアグラム**:
    - 非常にノード数が多い（100ノード超）、または複雑な依存関係を持つダイアグラムで、メモリリークや極端なパフォーマンス低下が発生しないこと。
- **不正な構文**:
    - 完全に壊れた構文を入力しても、IDE 自体がフリーズしたりクラッシュしたりせず、適切なエラーメッセージが表示されること。
- **空ファイル**:
    - 内容が空のファイルを開いてもエラーが発生せず、プレビューが適切に処理されること。
- **複数ファイル同時編集**:
    - 複数の Mermaid ファイルを同時に開き、それぞれが独立して正しくプレビュー・更新されること。
- **特殊文字**:
    - ラベル内に特殊文字（マルチバイト文字、HTML エンティティ等）が含まれていても、正しくレンダリングされること。
